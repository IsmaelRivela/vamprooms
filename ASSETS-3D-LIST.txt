BACKROOMS — Lista de assets 3D para buscar
==========================================

Formato obligatorio
-------------------
  • Archivo:     .glb  (glTF 2.0 binario, un objeto = un archivo)
  • Draco:       OK (compresión, como los de VAMPS Pharma)
  • Polígonos:   500 – 4 000 tris por prop (low-poly / PS1)
  • Escala real: 0,3 – 1,2 m de alto aprox.
  • Ejes:        Y arriba, base del objeto en Y = 0
  • Texturas:    embebidas en el GLB (512–1024 px)
  • Nombre:      kebab-case.glb  (sin espacios, sin símbolos +)

═══════════════════════════════════════════════════════════════
 COLOCACIÓN DE PROPS — editamos juntos estos JSON
═══════════════════════════════════════════════════════════════

Un archivo por sala en:
  public/room-layouts/la-tulipana.json
  public/room-layouts/copydad.json
  public/room-layouts/vamps-brand.json
  public/room-layouts/vamps-pharma.json
  public/room-layouts/vamps-back2school.json

Cada prop tiene:
  src          → ruta del .glb (ver lista arriba)
  position     → [X, Y, Z] relativo al CENTRO de la sala
                 X negativo = izquierda, X positivo = derecha
                 Z negativo = hacia pared norte, Z positivo = sur
  rotationDeg  → [X, Y, Z] en GRADOS (0–360)
  scale        → multiplicador extra (1 = normal)
  fitSize      → tamaño real objetivo en metros (ej. perchero 2.0, caja 0.4)
  yOffset      → subir/bajar fino si aún roza el suelo (ej. 0.02)

Ejemplo — mover el perchero de Copydad a la derecha:
  "position": [1.0, 0, 0]     ← más a la derecha
  "rotationDeg": [0, -90, 0]  ← girado

Workflow contigo + Cursor:
  1. Tú miras el juego y me dices: "el perchero más a la izquierda y más grande"
  2. Yo edito el JSON y tú recargas el navegador
  3. Repetimos hasta que quede bien

Para probar cambios: npm run dev → recarga la página (F5)
